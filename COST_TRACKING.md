# Cost Tracking Documentation

## Overview

The Editorial Assistant now tracks detailed token usage and cost data for all LLM API calls. This allows you to monitor spending and understand the economics of automated transcript processing.

## What's Tracked

For each project processed, the system records:

- **Input tokens**: Tokens sent to the LLM (system prompt + transcript)
- **Output tokens**: Tokens generated by the LLM (brainstorming, formatted transcript)
- **Total tokens**: Sum of input and output tokens
- **Estimated cost**: Calculated based on the model's pricing per 1M tokens
- **Model used**: Which specific LLM model processed the content (e.g., `gpt-4o-mini`)
- **Backend**: Which backend was used (`openai`, `anthropic`, `ollama`)

## Typical Costs

Based on processing with **gpt-4o-mini** (current primary backend):

### Standard Video (5-10 minutes)
- Transcript length: ~5,000-10,000 characters
- Total tokens: ~25,000-40,000
- **Estimated cost: $0.006-$0.010 per video**

### Long-Form Content (30-60 minutes)
- Transcript length: ~30,000-60,000 characters
- Total tokens: ~75,000-150,000
- **Estimated cost: $0.020-$0.040 per video**

### Digital Shorts (1-3 minutes)
- Transcript length: ~1,000-3,000 characters
- Total tokens: ~5,000-15,000
- **Estimated cost: $0.001-$0.004 per video**

## Cost Optimization Strategies

### 1. Use the Most Cost-Effective Model

The default **gpt-4o-mini** is already the most cost-effective OpenAI option. It provides excellent quality for transcript processing at ~4x lower cost than gpt-4o.

### 2. Consider Local Models (Ollama)

For very high-volume processing, local models are **free** but require:
- More powerful hardware
- Initial setup time
- May produce slightly lower quality results

Enable in `config/llm-config.json`:
```json
{
  "primary_backend": "local-ollama",
  "backends": {
    "local-ollama": {
      "enabled": true
    }
  }
}
```

### 3. Batch Processing

Processing multiple videos in a batch doesn't reduce per-video costs, but it:
- Reduces manual overhead
- Makes cost tracking easier
- Allows better budget planning

## Updating Pricing

If OpenAI or Anthropic changes their pricing, update the constants in `scripts/llm_backend.py`:

```python
# OpenAI pricing per 1M tokens
OPENAI_PRICING = {
    "gpt-4o-mini": {"input": 0.150, "output": 0.600},
    "gpt-4o": {"input": 2.50, "output": 10.00},
}

# Anthropic pricing per 1M tokens
ANTHROPIC_PRICING = {
    "claude-3-5-sonnet-latest": {"input": 3.00, "output": 15.00},
    "claude-3-5-haiku-latest": {"input": 1.00, "output": 5.00},
}
```

Always check official pricing pages:
- OpenAI: https://openai.com/api/pricing/
- Anthropic: https://www.anthropic.com/pricing

## Technical Details

### Files Modified
- `scripts/llm_backend.py` - Added `UsageMetrics` class and cost calculation
- `scripts/process_queue_auto.py` - Captures metrics and updates manifests
- `scripts/cost_report.py` - New script for cost analysis

### Data Flow
1. LLM API call returns usage data (tokens)
2. `llm_backend.py` calculates cost based on pricing table
3. Returns `UsageMetrics` object with all data
4. `process_queue_auto.py` stores metrics in manifest.json
5. `cost_report.py` aggregates data across all projects
